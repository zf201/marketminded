# DaisyUI Frontend Migration — Design Spec

**Date:** 2026-03-28
**Branch:** `refactor/daisyui-frontend`
**Goal:** Migrate the entire frontend from custom CSS to Tailwind + DaisyUI, consolidate JavaScript into Alpine.js components, and establish a reusable templ component library for consistent UX across all pages.

## Context

The current frontend uses hand-rolled CSS (164 lines), a monolithic vanilla JS file (1,889 lines), and 12 templ page templates with scattered inline styles. It works but has growing pains: duplicated chat initialization code across 3+ pages, inline styles throughout templates, no shared component abstractions, and inconsistent styling between pages.

The pipeline production board is the most polished page and the benchmark for quality. The rest of the app should match its level of polish.

## Decisions

- **Full restyle:** All 12 pages migrate to DaisyUI in one pass.
- **DaisyUI "business" theme:** Clean, professional, muted SaaS aesthetic.
- **Enhanced top nav + breadcrumbs:** Keep minimal top bar, add breadcrumbs for navigation context. No sidebar.
- **Alpine.js all-in:** Convert vanilla JS DOM manipulation into Alpine components. Keep Alpine + marked.js, no new frameworks.
- **Component library first:** Build shared templ components, then migrate pages to use them.
- **Pipeline UX tweaks:** Same core structure, restyled with DaisyUI + minor improvements (step progress indicator, cleaner content piece cards, DaisyUI modal for improve/proofread).
- **Backend untouched:** SSE event format unchanged. This is a pure frontend migration.

## Build Tooling

### Tailwind + DaisyUI Setup

Add Tailwind CSS CLI + DaisyUI plugin as a build step alongside templ generation.

**New files:**
- `tailwind.config.js` — Tailwind config with DaisyUI plugin, "business" theme, content paths pointing to `.templ` files
- `web/static/input.css` — Tailwind directives (`@tailwind base; @tailwind components; @tailwind utilities;`)
- `web/static/output.css` — Generated by Tailwind CLI (gitignored, built by `make`)

**Makefile changes:**
- `generate` target adds: `npx tailwindcss -i web/static/input.css -o web/static/output.css`
- New `watch` target for dev: runs templ generate + Tailwind in watch mode

**Deleted:**
- `web/static/style.css` — fully replaced by `output.css`

## Component Library

Shared templ components in `web/templates/components/`. Every page uses these instead of raw HTML + inline styles.

### Layout (`components/layout.templ`)

```go
// PageShell wraps page content with navbar + breadcrumbs + main container.
templ PageShell(title string, breadcrumbs []Breadcrumb) {
    // DaisyUI navbar
    // Breadcrumbs bar (if any)
    // <main class="container mx-auto p-4"> {children...} </main>
}

type Breadcrumb struct {
    Label string
    URL   string // empty for current page (last item)
}
```

- Navbar: DaisyUI `navbar` with "MarketMinded" branding left, Settings link right.
- Breadcrumbs: DaisyUI `breadcrumbs` component. Last item renders as text (current page), rest as links.
- Replaces current `Layout()` function.

### Card (`components/card.templ`)

```go
// Card renders a basic DaisyUI card.
templ Card(title string)

// CollapsibleCard renders a card with Alpine-powered toggle.
templ CollapsibleCard(title string, defaultOpen bool)

// StepCard renders a pipeline step card with status badge, tool pills area,
// thinking ticker slot, and output slot.
templ StepCard(step StepView)

// ContentPieceCard renders a content piece with status-colored left border
// and action buttons.
templ ContentPieceCard(piece PieceView)
```

- `Card`: DaisyUI `card card-bordered` with `card-title` and `card-body`.
- `CollapsibleCard`: Same card with `x-data="{open: defaultOpen}"` and `x-show="open"` on body. Toggle button in header.
- `StepCard`: Specialized for pipeline steps. Includes status badge in header, named slots for tool pills, thinking ticker, and output content. Status-specific border color via DaisyUI border utilities.
- `ContentPieceCard`: Status-colored left border (4px), platform/format badges, action buttons conditional on status. Uses `border-l-4` with color mapped from status.

### Badge (`components/badge.templ`)

```go
// StatusBadge renders a DaisyUI badge colored by status string.
templ StatusBadge(status string)
```

Status-to-variant mapping:
- pending → `badge-ghost`
- running → `badge-warning`
- completed → `badge-success`
- failed → `badge-error`
- draft → `badge-info`
- approved → `badge-success`
- rejected → `badge-error`
- generating → `badge-warning animate-pulse`
- abandoned → `badge-ghost`

### Form (`components/form.templ`)

```go
templ FormGroup(label string)      // DaisyUI form-control with label
templ TextInput(attrs InputAttrs)  // DaisyUI input input-bordered
templ TextArea(attrs TextAreaAttrs) // DaisyUI textarea textarea-bordered
templ Button(text, variant string) // DaisyUI btn btn-{variant}
templ SubmitButton(text string)    // DaisyUI btn btn-primary type=submit
```

### Modal (`components/modal.templ`)

```go
// Modal renders a DaisyUI dialog modal.
templ Modal(id string)
```

Uses the `<dialog>` element pattern with DaisyUI `modal` class. Opened via `document.getElementById(id).showModal()`. Replaces the current JS-built modal in `openContentModal()`.

### Chat (`components/chat.templ`)

```go
// ChatContainer renders a scrollable chat message list with Alpine streaming.
templ ChatContainer(config ChatConfig)

// ChatInput renders a textarea + send button with Cmd+Enter support.
templ ChatInput(placeholder string)

// ChatMessage renders an individual message bubble.
templ ChatMessage(role, content string)

type ChatConfig struct {
    ID        string // Alpine component ID
    StreamURL string // SSE endpoint
    SendURL   string // POST endpoint for user messages
    ProjectID int64
}
```

The `ChatContainer` uses `x-data="streamingChat({...})"` — the Alpine component handles all SSE streaming, thinking display, tool events, and message rendering. This single component replaces the 3 duplicated chat initializers.

## JavaScript Reorganization

### New Structure

```
web/static/js/
├── alpine-components/
│   ├── chat.js            # streamingChat — generic SSE chat
│   ├── pipeline.js        # pipelineBoard — step streaming, tool pills, run-next
│   ├── content-piece.js   # contentPiece — approve/reject/improve/proofread
│   └── collapsible.js     # collapsible — simple open/close toggle
├── renderers/
│   ├── markdown.js        # marked.js wrapper
│   ├── step-output.js     # Step type-specific output rendering
│   └── content-body.js    # Content type-specific body rendering
└── app.js                 # Entry: register Alpine components, DOMContentLoaded init
```

### Alpine Component Consolidation

**`chat.js` — `Alpine.data('streamingChat', opts)`:**

Replaces: `initProfileChat`, `initProfileSectionChat`, `initContextChat`, brainstorm inline script.

Config options:
- `streamURL` — SSE endpoint
- `sendURL` — POST endpoint for messages
- `onProposal(data)` — callback for proposal events (profile chat)
- `onDone()` — callback on stream completion
- `onContentWritten(data)` — callback for content_written events

Handles: message list, streaming state, thinking display, tool events, auto-scroll, Cmd+Enter send.

**`pipeline.js` — `Alpine.data('pipelineBoard', opts)`:**

Replaces: `initCornerstonePipeline`, `initProductionBoard`.

Handles: step streaming with tool pills and thinking ticker, run-next-step chaining, step progress indicator updates.

**`content-piece.js` — `Alpine.data('contentPiece', opts)`:**

Replaces: inline event delegation for approve/reject buttons, `openContentModal`.

Handles: approve/reject POST requests, opens DaisyUI modal for improve (with `streamingChat` inside), proofread request.

**`collapsible.js` — `Alpine.data('collapsible', opts)`:**

Simple `{open: defaultOpen, toggle()}`. Used by `CollapsibleCard` component.

### Renderers (Pure Functions)

These stay as stateless functions, just moved to their own files:
- `renderStepOutput(stepType, output)` — parses step JSON, renders sources/brief/issues/outline
- `renderContentBody(platform, format, body)` — dispatches to format-specific renderer
- `renderBlogPost`, `renderSimplePost`, `renderXThread`, `renderLinkedinCarousel`, etc. — unchanged logic, just relocated

### Loading

All JS files loaded as separate `<script>` tags in the layout template. No bundler needed at this scale. Alpine.js and marked.js stay as CDN scripts.

```html
<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
<script src="/static/js/renderers/markdown.js"></script>
<script src="/static/js/renderers/step-output.js"></script>
<script src="/static/js/renderers/content-body.js"></script>
<script src="/static/js/alpine-components/collapsible.js"></script>
<script src="/static/js/alpine-components/chat.js"></script>
<script src="/static/js/alpine-components/pipeline.js"></script>
<script src="/static/js/alpine-components/content-piece.js"></script>
<script src="/static/js/app.js"></script>
```

## Page Migrations

### Layout (`layout.templ`)

- Replace bare `<nav>` with DaisyUI `navbar` component via `PageShell`
- Add breadcrumbs bar below navbar
- Load `output.css` instead of `style.css`
- Load new JS file structure instead of monolithic `app.js`

### Dashboard (`dashboard.templ`)

- Project cards in responsive grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4`
- Each project uses `Card` component with name, description, link

### Project Overview (`project.templ`)

- Feature cards (Profile, Settings, Pipeline, Chat, Templates) use `Card` component
- Context items list uses DaisyUI `table` or card list

### Pipeline List (`pipeline.templ` — list view)

- Topic input form uses `FormGroup` + `TextInput` + `SubmitButton`
- Run cards use `Card` + `StatusBadge`

### Pipeline Board (`pipeline.templ` — board view)

- **New: Step progress indicator** — DaisyUI `steps` component at the top showing 6-step pipeline progress with completed/active/pending states. At-a-glance overview.
- Topic/brief section uses `CollapsibleCard`
- Step cards use `StepCard` component
- Content pieces use `ContentPieceCard` component
- Improve/proofread uses DaisyUI `Modal` component with `ChatContainer` inside
- Tool pills use DaisyUI `badge badge-sm` with appropriate colors
- Thinking ticker stays monospace scrollable — styled with Tailwind utilities

### Profile (`profile.templ`)

- 70/30 grid layout via `grid grid-cols-[7fr_3fr] gap-4`
- Chat side uses `ChatContainer` component
- Profile section cards use `Card` + `StatusBadge`

### Brainstorm (`brainstorm.templ`)

- Chat list uses `Card` components
- Chat page uses `ChatContainer` component
- **Delete inline script block** — behavior moves to `chat.js` Alpine component

### Context (`context.templ`)

- Uses `ChatContainer` with save action callback
- Same layout, cleaner markup

### Settings (`settings.templ`, `project_settings.templ`)

- All forms use `FormGroup` / `TextInput` / `TextArea` / `Button` components
- DaisyUI `select` for dropdowns (model selection, framework picker)

### Storytelling (`storytelling.templ`)

- Framework cards use DaisyUI `radio` + `collapse` components
- Selected state uses `card-bordered border-primary`

### Templates Manager (`templates_mgr.templ`)

- Add form uses form components
- Template list uses `Card` components
- HTML preview uses DaisyUI `mockup-code` component

### Project New (`project_new.templ`)

- Simple form with `FormGroup` + `TextInput` + `TextArea` + `SubmitButton`

## Deleted Artifacts

- `web/static/style.css` — replaced by Tailwind `output.css`
- `web/static/app.js` (monolithic) — replaced by `js/` module structure
- All inline `style="..."` attributes in templ files
- Custom utility classes (`.flex`, `.mb-1`, etc.) — Tailwind built-ins
- Custom component classes (`.card`, `.btn`, `.badge`, `.board-card-*`) — DaisyUI
- Inline `<script>` block in brainstorm.templ

## Preserved

- All `.templ` files (rewritten in place)
- `marked.js` CDN dependency
- `Alpine.js` CDN dependency
- All backend code (untouched)
- SSE event format (unchanged)
- Pipeline board UX patterns (collapsible cards, real-time streaming, tool pills, thinking ticker)
- Content type renderers (logic unchanged, just relocated)

## Out of Scope

- Backend changes (already refactored)
- New features or pages
- Schema changes
- Content type additions
- Dark mode / theme toggle (ship "business" only for now)
